1) BUG: UPLOADED TEASERS/SHORTS DON’T PLAY (WEB + PHONE)
Likely causes:
- Wrong Content-Type (served as application/octet-stream or text/plain)
- Missing Accept-Ranges / broken Range requests (Safari/iOS needs byte-range for MP4)
- CORS / auth redirect returning HTML (video tag downloads HTML login/404)
- Signed URL expiry / wrong URL
- Server returns 206/200 incorrectly, or blocks “Range” header
- File path / storage proxy not streaming

Tasks:
1.1 Verify API endpoint that serves video files (e.g. /api/files/:id, /uploads/*, or storage proxy).
- Ensure it supports Range requests:
  - If request has “Range: bytes=...”, respond 206 Partial Content with:
    Content-Range, Accept-Ranges: bytes, Content-Length, correct Content-Type (video/mp4)
  - If no Range, respond 200 with full file and correct headers.
- If using Express + static file:
  - Express static does NOT handle Range for some setups; implement manual streaming using fs.createReadStream with start/end.
  - Add “Accept-Ranges: bytes”.

1.2 Ensure correct Content-Type:
- MP4 => video/mp4
- WEBM => video/webm
- MOV => video/quicktime (better: transcode to mp4 if needed)
If stored in DB/file table, ensure mimeType column is saved from upload and reused on serve.

1.3 Ensure the response is NOT an HTML page:
- In Network tab, check response body is binary, not “Cannot GET …” or an auth page.
- If auth middleware blocks the file route, fix:
  - Either allow authenticated access properly (token/cookie included) OR generate a signed public URL for the <video src>.
  - Make sure the <video src> uses same-origin cookies if needed (no cross-origin mismatch).

1.4 Frontend <video> usage:
- Ensure we render:
  <video controls playsInline preload="metadata" ...>
- For iOS: playsInline helps; also avoid autoplay unless muted.
- Confirm src points to the correct served file URL.

1.5 Add a small diagnostic endpoint/log:
- Log file serve requests: id, range header, status, content-type, bytes served, user id.
- This should quickly confirm if Range is failing.

Acceptance:
- On Safari iPhone + Mac Safari/Chrome: newly uploaded teaser plays.
- Old teaser plays too.
- Network shows 206 on Range requests for mp4, correct headers.
